🔲 FASE 3 — SISTEMA DISTRIBUIDO (CRÍTICO)
================================================================================

Objetivo: arquitectura COGNIA MESH donde usuarios son nodos.
Regla de oro: lo episódico es SIEMPRE local y cifrado. Solo se comparte semántica.

PRINCIPIO DE DISEÑO:
  No P2P puro (fragmentación cognitiva). No cliente-servidor clásico.
  Replicación semántica selectiva con búsqueda federada bajo demanda.

ARCHIVOS A CREAR:
  network/mesh_node.py (NUEVO)
    - CogniaMeshNode: nodo en la red usando libp2p (py-libp2p)
    - publish_knowledge_delta(triples): publica solo triples nuevos
    - on_knowledge_received(): valida firma Ed25519 antes de integrar
    - federated_search(query_embedding, k, timeout_ms): busca en peers
      sin exponer memorias privadas

  network/crdt_graph.py (NUEVO)
    - CRDTKnowledgeGraph: G-Set (Grow-Only Set) para convergencia eventual
    - merge(remote_graph): unión idempotente, sin coordinador central
    - Un triple agregado nunca se elimina (solo invalida con metadatos)

  network/privacy.py (NUEVO)
    - privatize_embedding(vector, epsilon=1.0): privacidad diferencial
      añade ruido Laplaciano calibrado para compartir embeddings sin exponer datos
    - Capa 1 PRIVADO: episódico, nunca sale del dispositivo
    - Capa 2 SEMI-PRIVADO: resúmenes E2E cifrados para contactos autorizados
    - Capa 3 PÚBLICO: triples de conocimiento general anonimizados

TECNOLOGÍAS:
  - libp2p (py-libp2p) — más ligero que IPFS completo
  - CRDTs para consistencia sin coordinador
  - Privacidad diferencial (ε=1.0) para embeddings compartidos

🔲 FASE 4 — PRIVACIDAD Y SEGURIDAD
================================================================================

Objetivo: memorias cifradas E2E, control total del usuario sobre sus datos.

ARCHIVOS A CREAR:
  security/key_manager.py (NUEVO)
    - KeyManager: deriva clave maestra de passphrase con Argon2id
      (salt 32B, 3 iteraciones, 4 lanes, 64MB memory cost)
    - encrypt_memory(plaintext): AES-256-GCM con nonce aleatorio prepended
    - decrypt_memory(ciphertext): AES-256-GCM decrypt
    - Clave de sesión X25519 para datos semi-privados (E2E entre usuarios)

  security/secure_storage.py (NUEVO)
    - SecureEpisodicMemory: wrapper de EpisodicMemory que cifra/descifra
      transparentemente al leer/escribir
    - Flujo: [DB cifrada] → decrypt en RAM → [contexto] → [Ollama] → [borrar RAM]
    - NUNCA escribir contexto descifrado a disco
    - NUNCA loggear contenido de memorias privadas

REGLAS DE SEPARACIÓN DE DATOS:
  Capa PRIVADA  → AES-256-GCM, clave local derivada de passphrase
  Capa SEMI-PRIV → X25519 + ChaCha20-Poly1305, solo contactos autorizados
  Capa PÚBLICA  → sin cifrado, firmado con Ed25519, anonimizado

🔲 FASE 5 — ESCALADO DINÁMICO DEL MODELO
================================================================================

Objetivo: cambiar automáticamente entre modelos según recursos disponibles.

ARCHIVOS A CREAR:
  core/scale_manager.py (NUEVO)
    - ScaleManager: detecta nivel óptimo de operación en runtime
    - detect_level() → 1, 2 o 3 según RAM disponible + memoria count + peers
    - select_model(): retorna nombre del modelo Ollama para el nivel actual
    - watch(): hilo daemon que re-evalúa el nivel cada 60 segundos

NIVELES:
  NIVEL 1 — LOCAL PURO
    RAM < 4GB o < 100 memorias activas
    Modelo: llama3.2:3b-q4_K_M (2GB RAM)
    Storage: SQLite local, sin red
    Costo: $0

  NIVEL 2 — SEMI-DISTRIBUIDO
    RAM 4-12GB, 100-10.000 memorias, 1-50 peers
    Modelo: llama3.2:8b-q4_K_M o qwen2.5:7b (4-5GB RAM)
    Storage: SQLite + índice vectorial en disco
    Costo: $0 local, ~$5-20/mes si VPS pequeño

  NIVEL 3 — RED COMPLETA
    RAM > 12GB, > 10.000 memorias, > 50 peers
    Modelo: mixtral:8x7b-q3_K o API externa como fallback
    Storage: SQLite por usuario + índice vectorial federado
    Costo: ~$20-100/mes en VPS medianos

EDITAR:
  model_router.py
    - integrar ScaleManager para selección automática de modelo base
    - timeout adaptativo por nivel (nivel 1: 60s, nivel 2: 90s, nivel 3: 120s)
    - reemplazar cache FIFO simple por LRU real con hit rate tracking
